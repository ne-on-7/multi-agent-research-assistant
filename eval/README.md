# 📊 Evaluation Harness (RAGAS)

Offline, reproducible quality evaluation for the multi-agent RAG pipeline. It runs
a fixed question set through the **real** agents (Retriever → Web Researcher →
Synthesizer) and scores the answers with [RAGAS](https://docs.ragas.io), using our
own **Claude** model as the judge LLM and the same **all-MiniLM-L6-v2** embeddings
the app uses for retrieval.

## Run it

```bash
# from the project root, with your .env populated (ANTHROPIC_API_KEY etc.)
pip install -r requirements.txt -r requirements-eval.txt
python eval/run_eval.py
```

The harness ingests the bundled fixtures into an **isolated** vector store
(`data/qdrant_eval`) so it never touches your real document store. It prints a
metrics table, writes `eval/results.json`, and — if Langfuse keys are set — logs
the aggregate scores to Langfuse.

## Metrics

| Metric | What it measures |
|--------|------------------|
| **Faithfulness** | Are the answer's claims actually supported by the retrieved context (i.e. no hallucination)? |
| **Answer Relevancy** | Does the answer directly address the question? |
| **Context Precision** | Are the retrieved chunks relevant, and ranked well? |
| **Context Recall** | Did retrieval surface the information needed to cover the reference answer? |

## Files

- `dataset.json` — curated `{question, ground_truth}` pairs (mix of fixture-grounded and general questions).
- `fixtures/` — small source docs ingested before evaluation so the Retriever has real grounding.
- `run_eval.py` — driver: runs the pipeline, scores with RAGAS, writes results.
- `results.json` — generated output (per-sample scores + aggregate means).
